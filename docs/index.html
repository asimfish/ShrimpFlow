程序员的行为记忆与协作驱动系统

AI 编程助手最大的问题不是不够聪明，而是不够了解你。每次对话结束，上下文随即消失 —— 它不记得你昨天解决过什么问题，不了解你偏好的代码风格，也不知道你积累了哪些技术判断。

ShrimpFlow 记录你在终端、Git、OpenClaw、Claude Code 中的每一次操作和对话 session，将其可视化为日报、周报和完整的开发生涯画像。然后通过模式挖掘，将碎片化的行为数据归纳为你的 lifelong 行为模式数字孪生。

最终，这些行为模式被打包为 ClawProfile，一键下发给 Team Flow 驱动个性化项目开发 —— 你也可以在社区分享自己的虾行为模式，或导入其他大佬的 ClawProfile 来接管你的开发 workflow。

Pattern Mining Algorithms

Brain 层三维挖掘: 序列 x 时间 x 关联

Frequent Sequence

频繁序列挖掘 · windowSize=3 · minSupport=3

滑动窗口扫描事件序列，统计子序列出现频次，发现重复行为模式

git pull → code edit → git commit
x 47 times → confidence: 92%

Time Pattern

时间模式检测 · hourly[24] · weekly[7]

统计事件在 24h / 7d 维度的分布，发现周期性行为规律

10:00-11:00 code review
14:00-17:00 deep coding
→ Morning Review Pattern

Correlation Mining

关联规则挖掘 · window=10min · minConf=30%

10 分钟时间窗口内，发现跨数据源的事件关联

openclaw chat → pytest → commit
confidence: 85%

Parametric vs Non-Parametric Memory

与 OpenClaw-RL 互补的记忆机制

OpenClaw-RL

参数化记忆 · 行为训入模型权重

  • 行为 → 梯度更新 → 神经网络权重
  • 隐式存储，黑盒不可解释
  • 需要大量训练数据 + GPU 算力
  • 迁移需要 fine-tune 适配
  • 需要重新训练才能更新
VS

ShrimpFlow

非参数化记忆 · 结构化模式存储

  • 行为 → 模式提取 → 结构化 JSON
  • 显式存储，可审计可编辑
  • 少量数据即可提取模式
  • 可直接迁移、分享、组合
  • 实时更新，即时生效

互补关系: ShrimpFlow 的显式模式 → 反哺为 RL 的 reward shaping hints
