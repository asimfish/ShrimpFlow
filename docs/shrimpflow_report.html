Brain 层: 三大挖掘算法

频繁序列挖掘

Frequent Sequence Mining

滑动窗口 (size=3) 扫描事件序列，统计子序列出现频次

[git pull → code edit → git commit] × 47次 → 置信度 92%

minSupport = 3
windowSize = 3

时间模式检测

Time Pattern Detection

统计事件在 24h / 7d 维度的分布，发现周期性行为规律

每天 10:00-11:00 集中 code review → 晨间审查模式

hourly[24]
weekly[7]

关联规则挖掘

Correlation Mining

10 分钟时间窗口内，发现跨数据源的事件关联

openclaw chat → terminal pytest → git commit (置信度 85%)

window = 10min
minConf = 30%

参数化 vs 非参数化记忆

维度 OpenClaw-RL (参数化) ShrimpFlow (非参数化)
存储方式 行为 → 梯度更新 → 神经网络权重 行为 → 模式提取 → 结构化 JSON
可解释性 隐式存储，黑盒 显式存储，可审计可编辑
数据需求 大量训练数据 + GPU 算力 少量数据即可提取模式
迁移能力 需要 fine-tune 适配新任务 可直接迁移、分享、编辑
实时性 需要重新训练 实时更新，即时生效
互补关系 ShrimpFlow 的显式模式 → 反哺为 RL 的 reward shaping hints
